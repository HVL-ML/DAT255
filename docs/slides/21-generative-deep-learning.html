Learning distributions

For ML in general, we usually only to predict a single best value.

Requires there to be a functional relationship (one-to-one or many-to-one) between the the input features and the target

Learning distributions

Can free ourselves from the functional assumption by rather modelling the distribution that data came from

Example: Mixture density network predicting parameters of normal distributions

From this we can sample realistic new data points

Variational autoencoder

Twist: Let’s model distributions in latent space

This is the aim of the variational autoencoder (VAE).

  1. Encode input data into parameters of a probability distribution (for a normal distribution: mean and variance)
  2. Sample a point from this distribution
  3. Decode this point back to the input format

Variational autoencoder

Since encoder and decoder layers are nonlinear transformations, we can go from complicated input distributions to a normal (Gaussian) looking latent space and back

To force this latent space distribution, we modify the loss function:

Combine

  • reconstruction error (e.g. MSE), and
  • difference between the latent distribution and a normal distribution (measured by KL divergence)

Variational autoencoder

The latent space becomes continuous

We can sample along the different latent space dimensions to combine (or remove) concepts

Generative adversarial networks

Have two models work in tandem:

  • A generator that decodes random values into an image
  • A discriminator that takes in images and predicts whether it is real or made by the generator

Train so that

  • Generator gets better at producing realistic output
  • Discriminator gets better at distinguishing fake from real
