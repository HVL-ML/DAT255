Loss functions

Reminder: For all statistical modelling we need a measure of prediction quality

The loss function \(L\) must satisfy two conditions:

  • Differentiable*
  • Bounded below (\(L \geq 0\))

In this course we stick mostly to

  • Mean squared error (MSE) loss, for regression tasks

\[ \small L_{\mathrm{MSE}}(\boldsymbol{\hat{y}, y}) = \frac{1}{N} \sum_{i}^{N} (\hat{y}_i - y_i)^2 \] where \(N\) is the number of data points

*Again, at least piecewise differentiable

Loss functions

Reminder: For all statistical modelling we need a measure of prediction quality

The loss function \(L\) must satisfy two conditions:

  • Differentiable*
  • Bounded below (\(L \geq 0\))

In this course we stick mostly to

  • Mean squared error (MSE) loss, for regression tasks

  • Cross-entropy loss (or log loss), for classification tasks

\[ \scriptsize L_{\mathrm{log}}(\boldsymbol{\hat{y}, y}) = -\frac{1}{N} \sum_i^N \sum_k^K y_{i,k} \log\hat{y}_{i,k} \] where \(K\) is the number of classes

*Again, at least piecewise differentiable

Gradient descent

With the gradient in place, we take steps downward (along the negative gradient), towards the optimal solution:

\[ \small \boldsymbol{\color{teal}{\theta}}^{n+1} = \boldsymbol{\color{teal}{\theta}}^n - \eta \nabla \color{Purple}{L} \]

Here \(\eta\) is the learning rate

Momentum optimisation

Improve on regular gradient descent by keeping track of past gradients in a new term \(\boldsymbol{m}\):

\[ \small \begin{align} \boldsymbol{m}_n &= \beta \boldsymbol{m}_{n-1} - \eta \nabla \color{Purple}{L}(\color{teal}{\theta}_n) \\ \boldsymbol{\color{teal}{\theta}}_{n+1} &= \boldsymbol{\color{teal}{\theta}}_n + \boldsymbol{m}_n \end{align} \]

(behaves like a heavy ball rolling down a hill, where \(\boldsymbol{m}\) is the velocity)

Introduces a new hyperparameter \(\small\beta \in [0,1]\), which is called the momentum

  • \(\small\beta = 0\): “max friction”
  • \(\small\beta = 1\): “no friction”

Improve on regular gradient descent by keeping track of past gradients in a new term \(\boldsymbol{m}\):

\[ \small \begin{align} \boldsymbol{m}_n &= \beta \boldsymbol{m}_{n-1} - \eta \nabla \color{Purple}{L}(\color{teal}{\theta}_n) \\ \boldsymbol{\color{teal}{\theta}}_{n+1} &= \boldsymbol{\color{teal}{\theta}}_n + \boldsymbol{m}_n \end{align} \]

(behaves like a heavy ball rolling down a hill, where \(\boldsymbol{m}\) is the velocity)

Introduces a new hyperparameter \(\small\beta \in [0,1]\), which is called the momentum

keras.optimizers.SGD(learning_rate=0.01, momentum=0.9)

Nesterov accelerated gradient

Keep the concept of momentum, but compute the gradient slightly ahead:

\[ \small \begin{align} \boldsymbol{m}_n &= \beta \boldsymbol{m}_{n-1} - \eta \nabla \color{Purple}{L}(\color{teal}{\theta}_n + \beta \boldsymbol{m}_{n-1}) \\ \boldsymbol{\color{teal}{\theta}}_{n+1} &= \boldsymbol{\color{teal}{\theta}}_n + \boldsymbol{m}_n \end{align} \]

keras.optimizers.SGD(learning_rate=0.01, momentum=0.9, nesterov=True)

AdaGrad (Adaptive Gradient)

AdaGrad introduces an adaptive learning rate, which is adjusted independently for the different parameters

Adjustment is done by dividing by the sum of all past gradients

  • For parameters with steep gradient, learning rate is reduced quickly
  • For parameters with shallow gradient, learning rate is reduced slowly

RMSProp (Root Mean Square Propagation)

AdaGrad in practice:

  • Good: Less sensitive to choice of learning rate (since it’s adaptive)
  • Good: Fast convergence for simple problems
  • Bad: Slow (or no) convergence for difficult problems

RMSProp improves on AdaGrad by exponentially scaling down old gradients, before summing them

Good convergence for the difficult problems too

This scaling gives a new hyperparameter \(\rho\):

keras.optimizers.RMSprop(learning_rate=0.001, rho=0.9)

Learning rate and loss curves

Common to all optimisation methods is that we must choose a learning rate \(\eta\).

This affects the training progress:

Learning rate scheduling

Some options for the most efficient learning: (see notebooks)

  • Reduce \(\eta\) when learning stops:

    keras.callbacks.ReduceLROnPlateau(monitor='val_loss', factor=0.2, patience=5, min_lr=0.001)

  • Gradually reduce \(\eta\) for each step:

    keras.optimizers.schedules.ExponentialDecay(initial_learning_rate, decay_steps, decay_rate)
keras.optimizers.schedules.PolynomialDecay(initial_learning_rate, ...)

  • Change \(\eta\) by some other rule:

    class MyLRSchedule(keras.optimizers.schedules.LearningRateSchedule):

    def __init__(self, initial_learning_rate):
        self.initial_learning_rate = initial_learning_rate

    def __call__(self, step):
        return self.initial_learning_rate / (step + 1)

optimizer = keras.optimizers.SGD(learning_rate=MyLRSchedule(0.1))
