Diffusion models

When we talk about “diffusion models” we refer to the way they are trained.

The network architecture must be something that can de-noise an image, i.e. maintain original dimensions but improve the inherent concept

Good candidate: the U-Net

Ideally also want to encode the time step – hence the concat in the beginning

Training a diffusion model

Let’s now look at the forward and backward processes.

Forward diffusion process

Like for the VAE, we want a nicely structured latent space were we can create new data just by sampling from a normal distribution.

The forward process makes a corrupted image \(\boldsymbol{x}_1\) from an initial example \(\boldsymbol{x}_0\)

\[ \small \boldsymbol{x}_1 = \mathtt{noise\_rate}_1 \cdot \boldsymbol{\epsilon}_1 + \mathtt{signal\_rate}_1 \cdot \boldsymbol{x}_0 \]

where

  • \(\boldsymbol{\epsilon}_1\) are numbers sampled from a normal distribution (the noise)
  • noise_rate is a scaling for how much noise to add at this step
  • signal_rate is the remaining amount of the original image, at this step

Controlling the noise rate at each step is done by the diffusion schedule

Diffusion schedule

See best results if we add noise slowly in the beginning, using the cosine schedule
(rather than adding the same amount each time)

Cosine

The relation between noise and signal rates are given by

\[ \small \mathtt{noise\_rate}^2 + \mathtt{signal\_rate}^2 = 1 \]

Forward diffusion process

If being a little fancy about it, we can write the forward step

\[ \small \boldsymbol{x}_1 = \sqrt{\beta_1} \cdot \boldsymbol{\epsilon}_1 + \sqrt{1-\beta_1} \cdot \boldsymbol{x}_0 \]

(here with \(\beta\) being the noise variance) as a probability distribution \(q\):

\[ \small q(\boldsymbol{x}_1 | \boldsymbol{x}_{0}) = \mathcal{N}(\sqrt{\beta_1} \boldsymbol{x}_0, \beta_1 \boldsymbol{\mathrm{I}}) \]

where \(\mathcal{N}\) is the normal distribution (and \(\boldsymbol{\mathrm{I}}\) the identity matrix)

Trick: Since the sum of normal distributions is also normal, we can write the entire thing from start to \(t\) as

\[ \small q(\boldsymbol{x}_{t} | \boldsymbol{x}_{0}) = \mathcal{N}(\sqrt{\bar{\alpha}_t} \boldsymbol{x}_{0}, (1 - \bar{\alpha}_t) \boldsymbol{\mathrm{I}}) \]

with \(\small\bar{\alpha}_t = (1-\beta_1) \cdot (1-\beta_2) \cdot \ldots \cdot (1-\beta_t)\)

The factor \(\small\bar{\alpha}_t\) is basically the amount of original image remaining

Forward encoder

For the denoiser to be able to reverse the forward process, each step has to be small.

Need many steps in total, typically \(> 1000\).

Since we could write the output at an arbitrary step \(t\), we don’t have to run the forward process in sequence, but can train on the steps in random order

Reverse decoder

The distributions of the forward encoder \(\small q(\boldsymbol{x}_{t} | \boldsymbol{x}_{t-1})\) were easy

But computing reverse distribution \(\small q(\boldsymbol{x}_{t-1} | \boldsymbol{x}_{t})\) analytically is close to impossible.

Luckily we can approximate it with a deep learning model:

  • Model choice is independent of the diffusion process itself, as long as
    • Output dimensions are equal to input dimensions
    • We can (ideally) encode the time step information

Generating new samples

With the trained decoder model in place, we can finally generate new (unseen) images:

  • Step 0:
    Draw random pixel values from a normal distribution
  • Steps 1 to \(\sim\) 4000:
    Sequentially run the decoder to remove noise

Properties of diffusion models:

  • Easy to train
  • Superb quality of results
  • Slow to generate new samples
